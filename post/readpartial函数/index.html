readpartial函数

前面在调试一个接口时，自己在本地跑测试没有发现问题，部署到测试环境后测试发现报编码错误，Encoding::UndefinedConversionError: "\xE6" from ASCII-8BIT to UTF-8。自己也比较奇怪问题的发生，因为ruby在编码问题上其实做了很多的自动转换，所以碰到的并不多（当然如果你要是你用python2写过网页的抓取，就知道网页编码错误给新手带来的恐惧感😱 )。另外这个也与所运行的机器设置有关，这个在这就不探讨了。这里应该就是机器文件配置不一致，所以本地测试并没有发现。这里主要通过修改输出编码进行问题修复。

问题过程

查看代码定位到下面一段代码，

...
res_body = HTTP.get('http://news.baidu.com').body
res = ''

while tmp = res_body.readpartial
  res << tmp
end

res

代码上比较简单，使用了http这个gem包，然后读取请求的返回数据作为函数的返回值。而其中readpartial函数就是读取数据流的过程，查看文档可以发现

readpartial(maxlen) → string
Reads at most maxlen bytes from the I/O stream.

从stream中读取读取设定的字节，默认值应该是4096.

下面是示例：

irb(main):001:0> require 'http'
=> true
irb(main):002:0> a = HTTP.get('http://news.baidu.com')
=> #<HTTP::Response/1.1 200 OK {"Content-Type"=>"text/html;charset=utf-8", "Date"=>"Sun, 13 Aug 2017 07:13:38 GMT", "P3p"=>"CP=\" OTI DSP COR IVA OUR IND COM \"", "Server"=>"Apache", "Set-Cookie"=>"BAIDUID=2566632A5C67FBC740BA9306D2FCB7FD:FG=1; expires=Mon, 13-Aug-18 07:13:38 GMT; max-age=31536000; path=/; domain=.baidu.com; version=1", "Tracecode"=>["08188671830454104842081315", "08188390610572470282081315"], "Vary"=>["Accept-Encoding", "Accept-Encoding"], "Connection"=>"close", "Transfer-Encoding"=>"chunked"}>
irb(main):003:0> a.body
=> #<HTTP::Response::Body:3feb13b4e3b0 @streaming=false>

这里就出现了几个问题:

  1. 这是以字节模式读取数据，编码格式是ASCII-8BIT

    irb(main):005:0> c = a.body.readpartial
=> "<!doctype html>\n<html class=\"expanded\">\n<head>\n\n<!--STATUS OK-->..."
irb(main):006:0> c.encoding
=> #<Encoding:ASCII-8BIT>

  2. 设定maxlen后会截断数据

    当你设定了最大的读取长度后，函数会直接将数据截断，最明显的是存在汉字时，汉字为’\xE5…‘格式，如果长度到了设定值，会被直接截断，不论是不是完整的数据，因此会导致出错。尤其是当你边读边进行解析时，这个问题就会展现出来了。

解决办法

  1. 直接粗暴

    使用force_encoding('utf-8')方法将对象的编码转换成utf-8(这应该是目前最适用的编码格式)，这个方法非常直接，但是如果存在一些无法转换的字符，会导致函数转换错误。

  2. 修改配置

    1. 可以修改机器的配置，设置utf-8作为默认的编码格式；
    2. 在文件的开头加上#coding: utf-8设置文件的默认编码，不过新版本的ruby已经将utf-8作为默认的文件编码，所以现在几乎都忽略

  3. 使用’http’这个gem包

    根据’http’这个gem包的指引，可以发现，除了readpartial这个获取body信息方法外，可以直接调用to_s获取，获取到的格式为utf-8

    irb(main):027:0> a.body.to_s.encoding
=> #<Encoding:UTF-8>

最后
